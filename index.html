🎤 ภาพรวมโปรเจกต์ (Intro)

สคริปต์พูดแนะนำตัวโปรเจกต์:

"สวัสดีครับ โปรเจกต์นี้คือ DevBoard เป็นเหมือนเว็บบอร์ดสำหรับนักพัฒนา สร้างด้วย React ครับ จุดเด่นคือเป็น Web App ที่หน้าเว็บทำงานแบบไร้รอยต่อ (Single Page Application - SPA) คือการเปลี่ยนหน้าหรือค้นหาข้อมูลจะไม่มีการรีเฟรชหรือโหลดหน้าใหม่เลย ข้อมูลทั้งหมดเราจำลองดึงมาจาก API (JSONPlaceholder) และมีระบบกดหัวใจ (Favorite) ที่รีเฟรชหน้าแล้วข้อมูลก็ยังอยู่ครับ"

จุดเด่นที่ควรเน้น (Key Features)

  • Client-Side Routing: เปลี่ยนหน้าเร็วปรู๊ดปร๊าดผ่าน `react-router-dom`
  • Data Fetching: ดึงข้อมูลจาก External API กึ่งๆ Real-time เบื้องหลัง พร้อมสถานะ Loading/Error
  • Global State Management: จัดการข้อมูลที่ต้องใช้ร่วมกันหลายหน้าด้วย Context API
  • Data Persistence: เก็บข้อมูลไว้ใน Browser (LocalStorage) เพื่อไม่ให้หายตอนปิดหรือรีเฟรช

❤️ Context API (FavoritesContext)

คำอธิบายสำหรับพรีเซนต์: "ตรงนี้เป็นหัวใจของระบบ Favorite ครับ เราใช้ Context API คู่กับ localStorage (ความจำของเว็บบราวเซอร์) ข้อดีคือเวลาเปิดเว็บมา มันจะพยายามอ่านค่าที่เคยเซฟไว้ในเครื่องก่อนเสมอ และเราใช้ useEffect เป็นตัวคอยจับตาดูว่า ถ้าข้อมูล Favorites เปลี่ยนแปลง มันจะแอบเซฟลง localStorage อัตโนมัติเลยครับ มั่นใจได้ว่าข้อมูลไม่หายแน่นอน"

📄 src/context/FavoritesContext.jsx

import { createContext, useContext, useState, useEffect } from "react";

// 1. สร้าง context object
const FavoritesContext = createContext();

// 2. Provider component — ครอบ App ทั้งหมด
export function FavoritesProvider({ children }) {
  // อ่านค่าเริ่มต้นจาก localStorage (เช็คตอนแอปโหลดครั้งแรก)
  const [favorites, setFavorites] = useState(() => {
    try {
      const savedFavorites = localStorage.getItem('favorites');
      return savedFavorites ? JSON.parse(savedFavorites) : [];
    } catch (error) {
      return [];
    }
  });

  // Effect ทำงานทุกครั้งที่ state `favorites` มีการเปลี่ยนแปลง 
  // มันจะ sync ข้อมูลบันทึกทับลง localStorage แบบอัตโนมัติ
  useEffect(() => {
    localStorage.setItem('favorites', JSON.stringify(favorites));
  }, [favorites]);

  // ฟังก์ชันเอาไว้กดหัวใจ / เลิกกด
  function toggleFavorite(postId) {
    setFavorites((prev) =>
      prev.includes(postId)
        ? prev.filter((id) => id !== postId) // มีอยู่แล้ว -> เอาออก
        : [...prev, postId] // ยังไม่มี -> ใส่เพิ่มเข้าไป
    );
  }

  return (
    <FavoritesContext.Provider value={{ favorites, toggleFavorite }}>
      {children}
    </FavoritesContext.Provider>
  );
}

// 3. Custom hook ให้คนอื่นเรียกใช้ได้ง่ายๆ (ไม่ต้องส่ง Props)
export function useFavorites() {
  return useContext(FavoritesContext);
}

🎣 Custom Hook (useFetch)

คำอธิบายสำหรับพรีเซนต์: "อันนี้เป็น Custom Hook สไตล์เฉพาะตัวที่ผมสร้างแยกออกมาชื่อ useFetch ครับ จุดประสงค์คือลดความซ้ำซ้อนของโค้ด เพราะทุกเพจต้องดึงข้อมูลหมด เราจึงเอามารวมไว้ที่นี่ที่เดียว 它จะช่วยจัดการ 3 ตัวแปรหลักคือ: data (ข้อมูลที่ดึงได้), loading (สถานะวิ่งโหลด), error (ใช้ดักพัง) ทำให้ไฟล์อื่นๆ แค่พิมพ์บรรทัดเดียวก็ดึง API ได้แบบหล่อๆ เลยครับ"

📄 src/hooks/useFetch.js

import { useState, useEffect } from "react";

// Hook ทำหน้าที่ดึงข้อมูลจาก API, จัดการโหลดและดักจับ Error
function useFetch(url) {
  const [data, setData] = useState(null); 
  const [loading, setLoading] = useState(true); 
  const [error, setError] = useState(null); 

  // ฟังก์ชันสำหรับเรียก fetch ใหม่ (เพื่อใช้กับปุ่ม Reload ได้)
  const refetch = async () => {
    setLoading(true);
    setError(null);
    try {
      const res = await fetch(url);
      if (!res.ok) throw new Error("เกิดข้อผิดพลาดในการดึงข้อมูล");
      const result = await res.json();
      setData(result); // เก็บข้อมูลลง state data
    } catch (err) {
      setError(err.message); // เก็บ error
    } finally {
      setLoading(false); // ปิดตัวหมุนๆ Loading
    }
  };

  useEffect(() => {
    // โหลดข้อมูลอัตโนมัติเมื่อ Component ทำหรือ url เปลี่ยน
    refetch();
  }, [url]);

  return { data, loading, error, refetch };
}

export default useFetch;

📰 หน้าแสดงโพสต์และแบ่งหน้า (PostList)

คำอธิบายสำหรับพรีเซนต์: "นี่คือชิ้นส่วนหลักที่ใช้แสดงโพสต์ทั้งหมดครับ มี 3 ความสามารถย่อยอยู่ในตัวเลยคือ ระบบค้นหาแบบเรียลไทม์ (พิมพ์ปุ๊บหาปั๊บ), ปุ่มรีเฟรชข้อมูล, และ ระบบแบ่งหน้า (Pagination) สมมุติเราดึงข้อมูลมา 100 เรื่อง เราจะใช้คำสั่งหารจำนวนหน้า และใช้ฟังก์ชัน .slice() ของ JS เข้ามาหั่นอาร์เรย์มาโชว์แค่หน้าละ 10 เรื่องเท่านั้น เพื่อให้เบราว์เซอร์ไม่ค้างครับ"

📄 src/components/PostList.jsx

import { useState } from "react";
import PostCard from "./PostCard";
import LoadingSpinner from "./LoadingSpinner";
import useFetch from "../hooks/useFetch";

function PostList() {
  // ดึงข้อมูลง่ายๆ ด้วย useFetch Hook ของเราเอง
  const { data: posts, loading, error, refetch } = useFetch("https://jsonplaceholder.typicode.com/posts");
  const [search, setSearch] = useState("");
  const [currentPage, setCurrentPage] = useState(1); // เก็บหน้าปัจจุบัน

  if (loading) return <LoadingSpinner />;
  if (error) return <div>เกิดข้อผิดพลาด: {error}</div>;

  // 1. กรองโพสต์จากคำค้นหา (ถ้ามี) แบบ case-insensitive
  const filtered = posts ? posts.filter((post) =>
    post.title.toLowerCase().includes(search.toLowerCase()),
  ) : [];

  // 2. การแบ่งหน้า: แสดงหน้าละ 10 รายการ
  const itemsPerPage = 10;
  const totalPages = Math.ceil(filtered.length / itemsPerPage);
  
  // ตัด array เอาแค่โพสต์ในหน้าที่กำลังจะแสดง
  const currentPosts = filtered.slice((currentPage - 1) * itemsPerPage, currentPage * itemsPerPage);

  return (
    <div>
      {/* ส่วนหัว: ปุ่มรีโหลด */}
      <div>
        <h2>โพสต์ล่าสุด</h2>
        <button onClick={() => { setCurrentPage(1); refetch(); }}>
          🔄 โหลดใหม่
        </button>
      </div>

      {/* Input ช่องค้นหา */}
      <input
        type="text"
        placeholder="ค้นหาโพสต์..."
        value={search}
        onChange={(e) => {
          setSearch(e.target.value);
          setCurrentPage(1); // พิมพ์หาใหม่ ต้องเด้งกลับหน้าแรกเสมอ
        }}
      />

      {/* วนลูปโชว์ PostCard ทีละอัน */}
      {currentPosts.map((post) => (
        <PostCard key={post.id} post={post} />
      ))}

      {/* ส่วนควบคุมการแบ่งหน้า (Pagination) */}
      <div>
        <button 
          disabled={currentPage === 1}
          onClick={() => setCurrentPage(prev => Math.max(prev - 1, 1))}
        > ← ก่อนหน้า </button>
        
        <span>หน้า {currentPage} / {totalPages}</span>
        
        <button 
          disabled={currentPage === totalPages}
          onClick={() => setCurrentPage(prev => Math.min(prev + 1, totalPages))}
        > ถัดไป → </button>
      </div>
    </div>
  );
}

💬 โพสต์ย่อย (PostCard & CommentList)

คำอธิบายสำหรับพรีเซนต์: "โพสต์แต่ละอัน ถูกแยกออกมาเป็น <PostCard /> ครับ ไฮไลท์ของหน้านี้คือ ระบบ Lazy Fetching (การโหลดเมื่อจำเป็น) คอมเมนต์จะไม่ถูกดึงมาจากเน็ตเลยจนกว่าเราจะกดสั่ง 'ความเห็น' ระบบถึงจะดึง <CommentList /> ขึ้นมาทำงาน ช่วยลดภาระเซิร์ฟเวอร์มหาศาลครับ"

📄 src/components/PostCard.jsx

import { useState } from "react";
import { Link } from "react-router-dom";
import { useFavorites } from "../context/FavoritesContext";
import CommentList from "./CommentList";

function PostCard({ post }) {
  const { favorites, toggleFavorite } = useFavorites();
  const isFavorite = favorites.includes(post.id); // เช็คว่าโพสต์นี้โดนกดไลก์รึย้ง

  // State ควบคุมการเปิด/ปิดคอมเมนต์
  const [showComments, setShowComments] = useState(false);

  return (
    <div>
      <h3><Link to={`/posts/${post.id}`}>{post.title}</Link></h3>
      <p>{post.body}</p>

      <button onClick={() => toggleFavorite(post.id)}>
        {isFavorite ? "❤️" : "🤍"}
      </button>

      {/* ควบคุม State เปิดหน้าต่างคอมเมนต์ */}
      <button onClick={() => setShowComments(!showComments)}>
        {showComments ? "▲ ซ่อน" : "▼ ความคิดเห็น"}
      </button>

      {/* ถ้าแง้มดู (showComments = true) คอมโพเนนต์ถึงจะโหลดและเริ่มวิ่งดึง API */}
      {showComments && <CommentList postId={post.id} />}
    </div>
  );
}

📄 src/components/CommentList.jsx

import { useState, useEffect } from "react";
import LoadingSpinner from "./LoadingSpinner";

function CommentList({ postId }) {
  const [comments, setComments] = useState([]);
  const [loading, setLoading] = useState(true);

  // ดึงข้อมูลคอมเม้นก็ต่อเมื่อมันถูกเรียกใช้งานขึ้นจอแล้วเท่านั้น (Lazy Loading)
  useEffect(() => {
    async function fetchComments() {
      const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${postId}/comments`);
      const data = await res.json();
      setComments(data);
      setLoading(false);
    }
    fetchComments();
  }, [postId]); 

  if (loading) return <LoadingSpinner />;

  return (
    <div>
      <strong>ความคิดเห็น ({comments.length})</strong>
      {comments.map((comment) => (
        <div key={comment.id}>
          <div>{comment.name}</div>
          <div>{comment.body}</div>
        </div>
      ))}
    </div>
  );
}

📄 หน้ารายละเอียดโพสต์ (PostDetailPage)

คำอธิบายสำหรับพรีเซนต์: "เมื่อผู้ใช้คลิกเข้ามาเจาะลึกโพสต์ เราใช้ useParams() สกัดค่า ID ออกมาจาก URL (เช่น /posts/3 ได้เลข 3) แล้วเอาเลขนี้ไปยิง API ขอข้อมูลเฉพาะกระทู้นั้นมาแสดงพร้อมคอมเมนต์ถาวรครับ"

import { useState, useEffect } from "react";
import { useParams, Link } from "react-router-dom";
import { useFavorites } from "../context/FavoritesContext";
import CommentList from "../components/CommentList";

function PostDetailPage() {
  // สกัดพารามิเตอร์ ID โพสต์จาก URL โดยตรง
  const { id } = useParams(); 
  const { favorites, toggleFavorite } = useFavorites();
  
  const [post, setPost] = useState(null);
  useEffect(() => {
    async function fetchPost() {
      const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
      const data = await res.json();
      setPost(data);
    }
    fetchPost();
  }, [id]); // ผูกติดชีวิตไว้กับ ID ลิ้งก์เปลี่ยน ให้ขยับดึงตัวใหม่

  if (!post) return <div>กำลังโหลด...</div>;
  const isFavorite = favorites.includes(post.id);

  return (
    <div>
      <Link to="/">← กลับหน้าหลัก</Link>
      <h2>{post.title}</h2>
      <p>{post.body}</p>

      <button onClick={() => toggleFavorite(post.id)}>
        {isFavorite ? "❤️ ถูกใจแล้ว" : "🤍 ถูกใจ"}
      </button>
      
      {/* หน้าเจาะลึก เราโชว์คอมเมนต์ถาวรเสมอไม่หด */}
      <CommentList postId={post.id} />
    </div>
  );
}

🌟 หน้ารวมรายการโปรด (FavoritesPage)

คำอธิบายสำหรับพรีเซนต์: "หน้านี้ท้าทายมากครับ เพราะเรามีแต่ตัวเลข ID โพสต์ที่เซฟไว้ใน LocalStorage แต่ไม่มีข้อความโพสต์เลยท่าไม้ตายคือเราใช้ Promise.all() เข้ามาช่วยยิงสาด API พร้อมกันหลายๆ เส้นตามจำนวน ID ที่ถูกใจไว้ (ขบวนการดึงหมู่รวดเดียว) เพื่อเอามาประกอบร่างฝังโชว์ให้ในหน้านี้ครับ"

import { useState, useEffect } from "react";
import { Link } from "react-router-dom";
import { useFavorites } from "../context/FavoritesContext";

function FavoritesPage() {
  // ดึงตะกร้า ID (เช่น [1, 3, 5]) ที่กดถูกใจไว้มาจากท่อ Context
  const { favorites, toggleFavorite } = useFavorites();
  const [posts, setPosts] = useState([]);

  useEffect(() => {
    if (favorites.length === 0) return;

    // ระบบขบวนการดึงหมู่ ยิง API ซ้อนกันพร้อมกันทุกตัวในถุง ID 
    async function fetchFavoritePosts() {
      const results = await Promise.all(
        favorites.map((id) =>
          fetch(`https://jsonplaceholder.typicode.com/posts/${id}`).then((r) => r.json())
        )
      );
      setPosts(results); // ได้ครบทุกตัวลงเสร็จสรรพ
    }
    fetchFavoritePosts();
  }, [favorites]); 

  if (favorites.length === 0) return <p>ยังไม่มีโพสต์ที่ถูกใจ</p>;

  return (
    <div>
      <h2>❤️ โพสต์ที่ถูกใจ ({favorites.length})</h2>
      {posts.map((post) => (
        <div key={post.id}>
          <h3>{post.title}</h3>
          <p>{post.body}</p>
          <button onClick={() => toggleFavorite(post.id)}>
            ❤️ ยกเลิกถูกใจ
          </button>
        </div>
      ))}
    </div>
  );
}

🔍 ระบบค้นหาด้วย URL Params (SearchPage)

คำอธิบายสำหรับพรีเซนต์: "ที่หน้าค้นหาจังๆ ของเว็บ เราเปลี่ยนมาใช้ระบบ URL Search Params ครับ หมายความว่าเวลาเราพิมพ์หาคำว่า 'hello' ลิงก์เบราว์เซอร์จะเปลี่ยนเป็น /search?q=hello ข้อดีคือผู้ใช้สามารถก็อปลิงก์นี้ส่งให้เพื่อนกดเข้ามาดูผลลัพธ์แบบเดียวกันเป๊ะๆ ได้เลยครับ เป็นการค้นหาข้อมูลแบบ Advanced ที่เว็บบอร์ดใหญ่ๆ ใช้กัน"

📄 src/pages/SearchPage.jsx

import { useEffect, useState } from "react";
import { Link, useSearchParams } from "react-router-dom";
// (imports อื่นๆ ข้ามไปเพื่อความกระชับ)

function SearchPage() {
  // ดึงตัวจัดการ URL Params มาใช้ (?q=xxx)
  const [searchParams, setSearchParams] = useSearchParams();
  const query = (searchParams.get("q") || "").trim(); // อ่านค่า q จากโดเมน
  const [searchInput, setSearchInput] = useState(query); // เก็บค่าที่กำลังพิมพ์
  
  const { data, loading, error } = useFetch("https://jsonplaceholder.typicode.com/posts");
  const posts = data || [];

  // เวลากดปุ่มค้นหา อัปเดต URL Param แทน
  const handleSearch = (event) => {
    event.preventDefault();
    const trimmedQuery = searchInput.trim();
    if (!trimmedQuery) {
      setSearchParams({}); // ลบ param q ทิ้ง
      return;
    }
    setSearchParams({ q: trimmedQuery }); // URL จะเปลี่ยนเป็น ?q=คำค้น
  };

  // กรองโพสต์จากสิ่งที่มีใน URL (?q=xxx) 
  const filteredPosts = query
    ? posts.filter((post) => {
        const normalizedQuery = query.toLowerCase();
        return (
          post.title.toLowerCase().includes(normalizedQuery) ||
          post.body.toLowerCase().includes(normalizedQuery)
        );
      })
    : [];

  return (
    <div>
      {/* ฟอร์มและช่องกรอก */}
      <form onSubmit={handleSearch}>
        <input 
          value={searchInput} 
          onChange={(e) => setSearchInput(e.target.value)} 
        />
        <button type="submit">ค้นหา</button>
      </form>

      {/* โชว์ผลการค้นหาตาม filteredPosts */}
      {filteredPosts.map((post) => <PostCard key={post.id} post={post} />)}
    </div>
  )
}

1. Q: SPA (Single Page Application) ในโปรเจกต์นี้คืออะไร แตกต่างจากเว็บสมัยก่อนยังไง?

A: คือหน้าเว็บที่โหลดตัว HTML มาแค่ครั้งเดียว (ไฟล์เดียว) และใช้ JavaScript จำลองเปลี่ยนหน้าไปมาให้ความรู้สึกเหมือนเปลี่ยนหน้าเว็บโดยที่บราวเซอร์ไม่มีการโหลด/กระพริบจอเลยครับ

2. Q: ทำไมเราถึงต้องเอาแท็ก <BrowserRouter> ไปครอบตัว App ทั้งหมด?

A: เพื่อเปิดใช้งานระบบ Client-Side Routing ของ `react-router-dom` ทำให้ตัวเว็บฝั่งเบราว์เซอร์อ่านค่า URL และประมวลผลเปลี่ยน Component ได้ถูกพาธโดยไม่ต้องไปขอข้อมูลจากเซิร์ฟเวอร์หลังบ้านครับ

3. Q: ฟังก์ชันไลบรารี React.StrictMode ในไฟล์ main ช่วยเรื่องอะไร?

A: มันเป็นเครื่องมือตรวจสอบโค้ดแบบเข้มงวดของ React ครับ มันจะแอบรัน Component ของเราเบิ้ล 2 รอบแบบเงียบๆ ระหว่างช่วงที่กำลังเขียนโค้ด (Dev) เพื่อช่วยตรวจหา Side-effect หรือแจ้งเตือนสิ่งผิดปกติครับ

4. Q: ไฟล์ index.html แทบไม่มีโค้ดอะไรเลย แล้วหน้าเว็บทั้งหมดทำงานตรงไหน?

A: มันทำหน้าที่เป็นกระดานเปล่าครับ มีแค่แท็ก <div id="root"></div> เพื่อให้ JavaScript (React) เป็นตัวสร้างชิ้นส่วน UI แล้วนำไป 'ฝัง (Render)' ในกล่อง root กล่องนี้แทนครับ

6. Q: เครื่องหมายดอกจัน ในเส้นทาง <Route path="*" .../> มีการทำงานยังไง?

A: ใช้ดักจับเงื่อนไขอื่นๆ ทั้งหมด (Catch-all) หมายความว่าถ้าผู้ใช้พิมพ์ URL ยึกยือที่ไม่ตรงกับเส้นทางใดๆ ข้างบนเลย ตัวนี้จะทำงานและพาไปหน้า "404 Not Found" ครับ

7. Q: path="/posts/:id" ตัว :id เอาไปทำอะไร?

A: มันคือ Dynamic Segment ครับ ใช้เป็นตัวแปรรับค่าทาง URL เช่นเข้าลิ้งก์ /posts/123 โค้ดหน้าแสดงข้อมูลกระทู้ก็จะดูดค่า 123 ออกมายิงขอข้อมูลต่อครับ

8. Q: useParams() แตกต่างจาก useSearchParams() อย่างไร?

A: useParams() เอาไว้ดึงตัวแปรที่แฝงมาตามเส้นทางหลัก (เช่นจาก /posts/123 ดึง 123) ส่วน useSearchParams() เอาไว้ดึง Query ต่อท้ายหลังเครื่องหมาย ? (เช่น ?q=คำค้น ดึง คำค้น มา) ครับ

A: ข้อได้เปรียบใหญ่ที่สุดคือ ผู้ใช้สามารถไฮไลท์ก๊อปปี้ลิงก์ที่เห็นไปส่งให้เพื่อน แล้วเพื่อนเปิดมาก็จะเห็นคีย์เวิร์ดการค้นเดียวกันเป๊ะๆ ครับ

A: แท็ก <a> จะสั่งให้เบราว์เซอร์เริ่มโหลดหน้าเว็บใหม่ทั้งหมด แต่ <Link> จะขัดขวางจังหวะนั้น (prevent default) และสับเปลี่ยนคอมโพเนนต์ให้ทันที เพื่อไม่ให้เสียความลื่นของ SPA ครับ

13. Q: ทำไมต่อท้าย useEffect(..., []) เล็บอาร์เรย์จึงถูกเว้นไว้ว่างๆ?

14. Q: ถ้าเราลืมใส่วงเล็บอาร์เรย์ ต่อท้าย useEffect จะส่งผลเสียไหม?

A: มันจะส่งผลเสียอย่างหนักครับ มันจะถูกรันใหม่(ทำงาน) ทุกครั้งที่หน้าจอกระพริบวาดใหม่ ถ้ายิ่งข้างในมีโค้ดยิง API อยู่ เบราว์เซอร์อาจจะเกิด Infinite Loop (รันอนันต์) แล้วค้างได้

15. Q: ในระบบ Favorites... เราใส่ [favorites] ตัวเบอเร่อในวงเล็บอาร์เรย์ useEffect เพื่ออะไร?

A: เพื่อตั้งกล้องวงจรปิดจับตาดู state ชื่อว่า favorites ครับ เมื่อไรก็ตามที่คนใช้งานกดไลก์ ทำให้ state favorites เพิ่ม/ลด มันก็จะเรียกโค้ดข้างในทำงานซึ่งก็คือ เอาไปจดใส่สมุด localStorage ครับ

16. Q: เราจะแยกไปสร้าง useFetch ใหม่ทำไมสู้ใช้ fetch สดๆ ในโพสต์เลยไม่ได้หรือ?

17. Q: ตอนทำดึง API ทำไมต้องมีสถานะว่า loading: true/false

18. Q: ถ้าเน็ตกระตุก เซิร์ฟเวอร์ล่ม ทำยังไงไม่ให้หน้าขาว (App Crash)?

📌 หมวด 5: Context API และ Local Storage (21-25)

21. Q: ปัญหา Prop-Drilling ของ React แปลกๆ ที่ Context API ถูกสร้างมาเพื่อฆ่ามันคืออะไร?

22. Q: ทำไมระบบ "หัวใจ (Favorites)" ต้องใช้เทคนิค LocalStorage ทำไมตัวแปรปกติถึงเอาไม่อยู่?

A: เพราะเวลาผู้ใช้รีเฟรชหน้า (F5) หรือออกเบราว์เซอร์แล้วกลับมาใหม่ ตัวแปร State ทั้งหมดจะถูกลบเกลี้ยงเป็น 0 หมด LocalStorage ซึ่งเป็นฐานข้อมูลเบาๆ ฝังในบราวเซอร์จะติดแน่นอยู่ได้แม้ปิดคอมครับ

A: เพราะว่ากฎหมายของ LocalStorage บังคับว่า ห้ามเก็บข้อมูลรูปแบบอื่น ยกเว้น ตัวหนังสือก้อนเดียว (String) เราจึงต้องซิกแซกจับโครงสร้าง Array/Object แปลงเป็นข้อความก่อน

A: มันทำแปรสภาพกลับครับ โดยมันจะอ่านตัวหนังสือยาวๆ จาก LocalStorage คืนชีพกลับมาเป็น Array จริงๆ ให้ React นำไปใช้งานดึงโพสต่อได้ทันทีครับ

25. Q: "Provider" ใน Context คือคนจ่าย หรือผู้ให้ มันจ่ายอะไรให้ใครกันแน่?

A: จ่ายข้อมูลให้บรรดา Components ลูกทั้งหมดที่อยู่ภายใต้วงแขนมัน (ในไฟล์ App.jsx จะเห็นมันตั้งตระหง่านครอบ Navbar และ Routes) ทำให้ทุกหน้าได้สิทธิ์รับข้อมูลและอัพเดตกดหัวใจกันสบายๆ

📌 หมวด 6: เทคนิคเฉพาะ (Pagination & Searching) (26-30)

26. Q: ระบบหน้า Pagination เราใช้ท่าอะไรแบ่งกระบะข้อมูล มีการยิง API ถี่ๆ ตามจำนวนหน้าไหม?

A: ไม่ครับ เรายิงรวดเดียว 100 โพสเก็บใส่ตัวแปร แล้วใช้ท่า Client-side Pagination ด้วยตำสั่ง .slice() ของอาร์เรย์มาหั่นข้อมูลเฉพาะ 10 ตัวที่อยู่ในหน้านั้นออกมาโชว์ครับ

27. Q: สมการหาหน้า ทำไมตัวเลขต้องสวมคำสั่ง Math.ceil() เข้าไป?

A: ตัวนี้สำคัญมากครับ มันคือการสั่ง "ปัดเศษขึ้น" ไม่งั้นถ้าเรามีข้อมูล 23 โพสต์ โชว์หน้าละ 10 มันจะหารได้ 2.3 ทำให้ 3 โพสต์ท้ายตกหล่นหายเข้ากลีบเมฆ เมื่อถูกปัดขึ้นก็จะกลายเป็น 3 หน้าพอดีครับ

28. Q: การทำ "Lazy Fetching" กับ Comment มันช่วยโปรเจกต์นี้ตรงไหน?

A: เป็นการเขียนเงื่อนไขผูกให้ระบบทำงานโหลดคอมเมนต์ ก็ต่อเมื่อผู้ใช้ไปกดปุ่มกางแท็บดูเท่านั้น เราไม่ได้ผูกดึงรวบแต่ทีแรก ช่วยถนอมแบนด์วิดท์เซิร์ฟเวอร์แบบสุดโต่งและสปีดโหลดของเว็ปอย่างมาก

29. Q: ระบบค้นหาที่ทำด้วย .filter() ต้องใส่คำสั่ง `.toLowerCase()` แทรกทุกๆ ที่เพื่อเหตุผลกลใด?

A: เพื่อทำ Case-insensitive Search ครับ ทำให้แม้ว่าผู้ใช้พิมพ์ตัวพิมพ์เล็ก (a) แต่ในฐานข้อมูลเป็นพิมพ์ใหญ่ (A) ก็ยังตีความได้ว่ามันเข้ากัน เจอและแมตช์ได้ 100%

30. Q: จากที่ทำ 67160003 DevBoard ขึ้นมา จุดแข็งที่สุด และความลับที่คนนอกไม่ค่อยรู้อย่างหนึ่งคือการจัดการ State ที่คุณภูมิใจคืออะไร?

A: นี่คือไม้ตายสุดท้าย! ความลับคือการรวมเอา 3 เรื่องอย่าง Custom Hook ลดความน่าปวดหัวของ API ผนวกเข้ากับ Context API และผสานเข้าเบราเซอร์อย่าง LocalStorage โค้ดทั้งหมดจะโยงใยเชื่อมถึงกันได้สมบูรณ์ และทำให้เว็บนี้ไม่ต้องพึ่งพาเซิรชเอนจิ้นหลังบ้านแพงๆ เลยครับ!

📌 หมวด 7: รูปแบบการทำงานและการเรนเดอร์ (React Lifecycle) (31-35)

31. Q: ทำไมใส่ setState ในตัว Component โดยตรง (นอก useEffect หรือฟังก์ชัน Event) ถึงทำให้เว็บค้าง?

A: เพราะมันจะเกิด Infinite Loop (วนลูปนรก) พอโหลดมาเจอคำสั่งเซ็ต State -> ค่าเปลี่ยน -> จออัปเดตเริ่มต้นพ่นโค้ดใหม่ -> เจอคำสั่งเซ็ต State อีก -> จออัปเดต วนไปเรื่อยๆ จนเบราว์เซอร์รับไม่ไหวครับ

32. Q: เวลาเอา Array มา .map() สร้าง Component วนลูป ทำไมต้องใส่ key ถ้าไม่ใส่จะมีผลอะไร?

A: key เป็นเหมือนบัตรประจำตัวประชาชนให้ React รู้ว่าชิ้นไหนคือชิ้นไหน ถ้ามียูสเซอร์ลบโพสต์อันที่สองออก React จะได้หันไปลบถูกชิ้น ไม่ต้องพังกระดานเคลียร์วาดใหม่ทั้งหมดให้หนักเครื่องครับ

33. Q: ทำไมในโปรเจกต์นี้ถึงใช้ key={post.id} แทนที่จะใช้ key={index} (ลำดับการวนลูป 0,1,2...)?

A: เพราะลำดับ index เปลี่ยนแปลงได้ตลอดเวลาถ้าข้อมูลถูกขยับหรือลบทิ้ง ทำให้หน้าตาและการอัปเดต UI ผิดเพี้ยน การใช้ ID ของฐานข้อมูลที่เป็น Unique ตายตัวจะลบจุดอ่อนนี้ 100%

34. Q: คำที่สายโปรแกรมเมอร์เรียกว่า "Component Mount" มีความหมายว่าอะไร?

A: คือวินาทีแรกสุดที่ชิ้นส่วนคอมโพเนนต์นั้น "ถูกนำมาปูวางแปะลงบนหน้าจอ" ซึ่งเป็นเวลาทองที่เราต้องเขียน useEffect สั่งดึง API เตรียมของให้พร้อมใช้งานครับ

35. Q: Virtual DOM ที่เขาบอกว่าเป็นจุดแข็งของ React คืออะไร?

A: มันคือร่างโคลนนิ่งของหน้าจอที่ React แอบจำลองไว้ในแบ็คกราวน์ เวลาสั่งเเก้เไข มันจะแก้ในร่างโคลนแล้วเอามาเทียบกับจอจริง ถัดมาจะจิ้มแก้ไข "เฉพาะจุดย่อยนิดเดียว" ที่เปลี่ยนไปจริงๆ ทำให้เว็บลื่นไหลไม่กระตุกครับ

📌 หมวด 8: ประสบการณ์ผู้ใช้และการประมวลผล (UI/UX) (36-40)

36. Q: ในการเขียน UI ของ React ทำไมถึงห้ามใช้ class="btn" เหมือนที่เคยเรียนมา แล้วต้องมาพิมพ์ className="btn" แทนตลอด?

A: เพราะคำว่า class มันดันบังเอิญไปซ้ำกับคำสงวน (Reserved Word) ของภาษา JavaScript ที่เอาไว้สร้าง Class OOP พอดีครับ กฎหมู่ออกแบบเลยบังคับให้หนีไปใช้ className แทนครับ

37. Q: การทำหน้าดักทาง <NotFoundPage /> (โชว์เตือน 404) สำคัญยังไงต่อคะแนน UX?

A: ถ้าไม่ทำกรงดักไว้ เวลา user พิมพ์ URL มั่วซั่วหรือลิงก์เก่าตาย จะเจอจอเบราว์เซอร์สีขาวแหงก (หรือหน้า Default โง่ๆ) ตกใจกลัวปิดหนีเบย การดัก 404 สุภาพๆ พร้อมปุ่มโยนกลับหน้าแรก เป็นการดูแลจิตใจคนใช้ครับ

38. Q: ไอคอน Loading Spinner วงกลมหมุนๆ ทำไมถึงหมุนได้ลื่นปรื้ดทั้งที่เวลาดึง API เครื่องมักจะกระตุกค้าง?

A: เป็นการแยกสมองการทำงานครับ! สปินเนอร์เราโหลดด้วยอนิเมชั่นของ CSS @keyframes โดยตรง การ์ดจอรับเหมาไปทำ เลยไม่สะทกสะท้านกับสมองหลัก (Main Thread JavaScript) ที่ฝืนกำลังดึงข้อมูลข้ามทวีปครับ

39. Q: เวลากดปุ่ม Favorite ทำไมหัวใจถึงแดงเปล่งประกายขึ้นมาทันที ไม่ต้องโหลดงึกๆ งักๆ แลกข้อมูลกับเซิร์ฟเวอร์?

A: เพราะการเปลี่ยนสีไอคอนถูกควบคุมด้วย Local State ระดับไคลเอนต์ครับ พอกดคลิก เราอัปเดตทาสีในฝั่งหน้าเว็บทันทีในเสี้ยววิ (Optimistic UI Approach) แล้วเบื้องหลังค่อยแอบไปเซฟให้เงียบๆ

40. Q: การแตกโครงสร้างออกมาเป็น Components เล็กๆ เอาเป็นเอาตายแบบนี้ มันดีกว่าเขียนรวมในก้อนเดียวหรอ?

A: ลดการกระเพื่อมครับ! ถ้ารวมกัน พอฟิลเตอร์คีย์เวิร์ดเดียวหน้าเว็ปทั้งหมดพันแถวจะรีแรนเดอร์กระพริบใหม่ทั้งหมด ทรัพยากรหมดเกลี้ยง ถ้าเรายิ่งซอยย่อย การอัปเดตก็จะเกิดเฉพาะปะทะในกล่องเล็กๆ แค่กล่องเดียว

📌 หมวด 9: ควบคุม JavaScript ขั้นสูงเสมือน React (41-45)

41. Q: ลูกศรเก๋ๆ => (Arrow Function) ที่ใช้ทั้งโปรเจกต์นอกจากสั้นแล้ว ดีกว่า function แบบเก่าตรงไหน?

A: โคตรวิเศษตรงที่มันไม่มีทิศทางของคำว่า this เป็นของตัวเองครับ มันจะสืบทอดสายเลือด this มาจาก Component ที่ห่อหุ้มมัน ทำให้การเรียกตัวแปรไปมาไม่ผิดเพี้ยนเลย

42. Q: ทำไมหน้าค้นหา(Search) เราถึงควัก .filter() มาใช้แทนที่จะใช้ .map() ธรรมดา?

A: .map() คือการแปรรูป (เช่น โยนแป้งไป 10 ก้อน ได้ขนมปังหน้าตาใหม่ 10 ก้อน) แต่ .filter() คือการเอาร่อนตะแกรง ดึงขยะทิ้ง โยนสิบก้อนอาจจะได้ออกมารอดการค้นหาแค่ 2 อันครับ

43. Q: เครื่องหมายจุดสามจุด ...prev (Spread Operator) ตอนใช้ setFavorites มีคุณงามความดีคืออะไร?

A: คุณสมบัติกางตำราแผ่ครับ! มันคือการระเบิดอาเรย์เก่าที่เคยกดไลก์ไว้ กระจายมาปูพื้นเรียงไว้ก่อน แล้วค่อยหยิบไอเท็มดวงใหม่ไปประกบท้าย ทำให้ข้อมูลของเด่าที่เรารักไม่สูญหายไปไหนครับ

44. Q: const posts = data || [] ไอ้เส้นตั้ง 2 เส้นคู่กันเอาไว้ทำเพื่ออะไร?

A: มันเป็นเทคนิค "เตียงสำรอง (Fallback)" ครับ สมมุติยิง API ยังไม่เสร็จ data จะเป็นก้อนอากาศเน่า (null) ถ้า React เอาไปเข้าเครื่องปั่นมันจะงงและพัง เราเลยดักทิศว่า ถ้าเน่า ให้ดึง [] (กล่องเปล่า) ไปใช้แทนพลางๆ ก่อน

45. Q: รูปประโยคจังๆ แบบ { error ? <ErrorMsg /> : <DataScreen /> } ใน React เรียกว่าอะไร?

A: มันคือ Ternary Operator (เงื่อนไข If-Else ฉบับย่อ) คือถ้าจริงเเบะซ้าย (หลัง '?') ถ้าโกหกเบียดขาว (หลัง ':') ซึ่งเป็นท่าไม้ตายในการสลับหน้าจอ UI สวรรค์หรือนรกของคนเขียน React ทั่วโลกครับ

📌 หมวด 10: การเตรียมแพคเก็จพร้อมวางขาย (Architecture & Deployment) (46-50)

46. Q: โฟลเดอร์มหึมา node_modules หนักเป็นกิ๊ก ทำไมเวลาส่งงานอาจารย์ หรือดันโค้ดขึ้น GitHub เราถึงต้องบลานซ์ลบมันเสมอก่อน?

A: เพราะเราสามารถร่ายคาถาสั่ง npm install ชุบชีวิตเสกมันกลับมาครบทุกหยดได้เลยแบบอัตโนมัติ โดยอิงทะเบียนประวัติจาก package.json การนำขยะขึ้น Git จึงเป็นเรื่องน่าอับอายขายขี้หน้าครับ

47. Q: สิ่งมีชีวิตที่เรียกว่า `package.json` เป็นหัวใจตรงไหน?

A: เป็นศูนย์บัญชาการครับ เก็บชื่อโปรเจค เก็บรายชื่ออาวุธ(เครื่องมือ/ไลบรารี่)ทั้งหมดที่ใช้ในเว็ป และเก็บโค้ดเส้นทางลัด (scripts) ว่าถ้าจะรันเข้าเซิฟหลักต้องพิมพ์อะไร

48. Q: คุณเอาสิ่งที่เขียนในไฟล์รหัสประหลาด .jsx ไปเปิดใน Google Chrome มันจะรันขึ้นมาสวยๆ ขย้ำได้เลยไหม?

A: พังยับเยินจ้า! โครมและเบราว์เซอร์ทุกเจ้าไม่มีใครอ่าน jsx เข้าใจเลยครับ มันจึงต้องมีตัวแปลภาษา (เช่น Vite+Babel) ดูดข้อความแปลกๆ บีบรัดเป็น JavaScript ออริจินัลนัวๆ ให้เบราว์เซอร์สูบดมครับ

49. Q: โปรเจกต์นี้เริ่มต้นด้วยเครื่องทุ่นเเรง Vite มันดีกว่าชาวบ้านชาวช่อง (เช่น Create React App เดิม) อย่างไร?

A: Vite ดีเลิศประเสริฐสุดเพราะ "เร็วทะลุขอบนรก" ครับ มันรีดโค้ดเปลี่ยนเฉพาะก้อนที่แก้ไขเบ็ดเสร็จในเสี้ยววินาที ไม่ต้องปิงประมวลผลวาดโปรเจกต์ใหม่ทุกครั้งเหมือนเครื่องยุคหินครับ

50. Q: ก่อนจะเสกขึ้นโฮสติ้งเป็น Production จริงจัง โฟลเดอร์ลับ dist/ จะโผล่ขี้นมา โฟลเดอร์นั้นมีไว้ทำไม?

A: มันย่อมาจาก Distribution ครับ! มาจากตอนที่เราพิมพ์ npm run build ระบบจะจับเอา src โค้ดทั้งหมดมาย่อยสลายความเวิ่นเว้อเหลือแต่ความเพอร์เพคที่เบาหวิว โกยทิ้งเป็นไฟล์ HTML/CSS/JS บริสุทธิ์ผุดผ่อง พร้อมนำไปวางที่เซิฟเวอร์แจกชาวบ้านแล้วครับ
